Gaze360踩坑记录

记录一下为了跑Gaze360在真机装Ubuntu踩的坑

Gaze360踩坑记录

起因

之前一直在用kali虚拟机做些ctf的题，感觉完全够用，然后anaconda的环境配置在windows真机，但是最近因为要看Gaze360，想用anaconda的环境而不是Colab之类的云环境，不得不在linux上起一遍anaconda

Ubuntu注意事项

SATA Operation - AHCI

Ubuntu只认AHCI的硬盘存储模式，我的电脑默认是RAID，如果在装Ubuntu之前不把Windows的安全引导打开，会导致装Ubuntu之后无法正常进入Windows

谨慎安装显卡驱动

曾跟着搜到的帖子安驱动导致Ubuntu一进桌面就卡死然后开始报错，在Activity->Additional Driviers管理驱动是一个更稳妥的办法，关于安装驱动后某一显示器/外接显示器的白平衡突然失调，可以在Settings->Color选择正确的Color Profile即可

为终端命令启用代理

安装proxychains4，在Settings->Network中设置的代理仅用于浏览器

为终端启用自动补全

安装zsh-autosuggestions

删除软件

sudo apt remove xxx
sudo apt remove --auto-remove xxx
sudo apt purge xxx

Conda注意事项

不要在VMware做深度学习

VMware虽然有Nvidia提供的GPU虚拟化，但需要setup manully，性能折扣情况不明，但够麻烦，所以我直接装真机Ubuntu，与Windows共存

安装Conda后默认进入Base

安装Conda后，每次打开终端都会默认进入Base环境，表现为提示符前面有一个(base)，如果不注意这一点，会导致虚拟环境和真机环境的混乱，调整conda设置即可使得打开终端时不进入Base环境

在Conda下使用JupyterNotebook

在Conda环境下使用Jupyter，如果在真机手动apt install jupyter会出现如redirect file显示无法访问之类的问题

TF1.15最高支持到Py3.7

现在的Conda默认使用Py3.9的Kernel，会找不到TF1.15的package，而很多老项目用到的都是TF1.15,所以需要注意Python版本

Conda换源

vim ~/.condarc
